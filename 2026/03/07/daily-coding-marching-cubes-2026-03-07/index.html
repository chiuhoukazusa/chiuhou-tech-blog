算法原理

Marching Cubes 核心思想

Marching Cubes 将3D空间划分为等大的立方体体素。对于每个体素，检查其8个角点的标量场值：

  • 值 < 0（等值面内部）：角点标记为1
  • 值 ≥ 0（等值面外部）：角点标记为0

8个角点的二进制状态组合成一个0-255的索引，查找预定义的三角形表，得到该体素对应的三角形边。

技术总结

学到的技术点

  1. Marching Cubes 三角形查找表：256种配置是将256个二进制索引预先计算好三角形方案，避免运行时复杂判断。标准实现通常使用 Paul Bourke 或 Lorensen & Cline 1987 论文的原始查找表。

  2. Smooth Union（软并集）：参数 k 控制融合半径，值越大，过渡越平滑；值越小，越接近硬并集。k=0.5f 产生有机的融合效果。

  3. 数值梯度 vs 解析梯度：对于复杂的隐式函数，数值梯度（中心差分）更通用，只需要一个额外的微小步长 eps。解析梯度在简单函数时更快，但需要手动求导。

  4. Z-Buffer 正交投影：正交投影对于展示3D几何形状很直观，没有近大远小的透视变形，适合技术展示场景。

遇到的坑

  • C++ 成员变量初始化顺序警告：-Wreorder 警告提示 zbuffer 的初始化依赖于 W 和 H，但声明顺序导致 zbuffer 先初始化。修复方法：将 W, H 声明移到 zbuffer 之前。

可扩展方向

  • 添加透视投影
  • 实现 Dual Contouring（更好保持边缘特征）
  • 使用 OpenMP 并行化 Marching Cubes
  • 实时动画（时间变化的标量场）
  • 导出 OBJ/PLY 格式
